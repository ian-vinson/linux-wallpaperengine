ISSUE-04: PKG binary package format — multi-version support          [PARTIAL]
  CONFIRMED FORMAT TABLE (from reverse engineering across 15+ wallpapers):
    PKGV0001: unpadded, relative offsets  (Senran Kagura GIF, 4 entries)
    PKGV0005: unpadded, relative offsets  (Sky of Zelda 1977866140, 42 entries)
    PKGV0008: unpadded, relative offsets  (Scarlet Witch, 60 entries)
    PKGV0013: unpadded, relative offsets  (Music Cat 2149068390, 18 entries)
    PKGV0018: unpadded, relative offsets  (Maxwell Cat 2916644233, 42 entries)
    PKGV0020: unpadded, relative offsets  (Gengar 2 3244466773, 100 entries)
    PKGV0021: TWO VARIANTS — self-validation required:
              PADDED:   4-byte aligned names, absolute offsets
              UNPADDED: packed, relative offsets (Shiina Mashiro, Gwen Stacy)
    PKGV0022: unpadded, relative offsets  (Cyberpunk Lucy, 155 entries)
    PKGV0024: unpadded, relative offsets  (Ocarina, 873 entries)

  CONCLUSION: Unpadded spans PKGV0001–0024. PKGV0021-padded is the ONLY exception.

  FIX — self-validating detection:
    1. Parse using UNPADDED format.
    2. Check: data_start + last_rel_offset + last_size == file_size
    3. If yes → unpadded correct.
    4. If no → re-parse PADDED. If neither validates → warn, use padded.

  VALIDATION CHECKSUMS:

ISSUE-10: perspectiveoverridefov misapplied on orthographic scenes  [PENDING / TIER 4]
  CONFIRMED on 5+ wallpapers: Shiina Mashiro, Neon Cafe, Stardew Valley,
    Windows 7 Remastered, Bunk, JDM Anime, Mono No Aware (giant cat).
  Root cause: lwe applies perspectiveoverridefov even when
    orthogonalprojection is set. When both are present, the scene is 2D
    orthographic — perspectiveoverridefov should be IGNORED.
  Symptom pattern:
    - clearcolor visible (grey/navy) instead of GL black → lwe is rendering
    - Positioned objects at wrong scale/location
    - Objects at high parallaxDepth massively magnified (Mono No Aware cat)
    - "stuck in top-right corner" on wide scenes
  Fix: In CScene.cpp, check for orthogonalprojection block. If present,
    set orthographic projection using those dimensions and skip
    perspectiveoverridefov entirely.
  File: src/WallpaperEngine/Render/Wallpapers/CScene.cpp
  Test wallpapers (all have orthogonalprojection + perspectiveoverridefov):
    Gengar (3300031038): {2560×1440}, pFOV=30
    Shiina Mashiro (3351163962): {3840×2160}, pFOV=30
    Windows 7 Remastered (3514379430): {1920×1080}, pFOV=90

===================================================================
PKG FORMAT REFERENCE
===================================================================

HEADER (all versions, always 16 bytes):
  [uint32 magic=8][char[8] version="PKGVnnnn"][uint32 entry_count]

UNPADDED FORMAT (ALL versions except PKGV0021-padded):
  [uint32 name_len][char* name][uint32 rel_offset][uint32 rel_size]
  rel_offset: relative to data_start
  data_start = 16 + sum(4 + name_len + 8) for all entries
  Validation: data_start + last_rel_offset + last_size == file_size

PADDED FORMAT (PKGV0021-padded ONLY):
  [uint32 name_len][char* name][PADDING to 4-byte][uint32 abs_offset][uint32 abs_size]
  abs_offset: absolute from file start
